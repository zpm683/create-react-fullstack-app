## ReactNative

Native App （原生应用）
由移动设备厂商自身提供的 SDK 或建议的开发方式来进行开发。
● 优点：
用户的交互体验非常好，性能最好
贴近底层，操作更加的流畅
便于调用硬件设备
不依赖于网络
× 缺点：
同一个项目，针对不同的手机操作系统都要进行独立的开发
测试非常麻烦, 每一次修改都需要重新打包
学习成本高
更新缓慢

Web App （H5 应用）
使用 HTML/CSS/JS 的开发方式来进行开发。
● 优点：
调试方便
无需主动推送升级
开发成本低（对于前端程序员）
基于浏览器，跨平台
× 缺点：
重度依赖于网络
无法调用系统底层
与原生相比交互体验差

Hybrid App （混合应用）
使用 web app 的技术进行开发，再封装成类似原生的 App。
● 优点：
开发上比原生简单
功能上比 H5 应用全面
× 缺点：
性能不如原生 App 稳定
