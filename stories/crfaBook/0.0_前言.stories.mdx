### 献给兴趣使然的人

## 前言

随着智能移动终端的普及和分布式互联网技术的发展，人们越来越多的通过浏览器进行学习，工作，娱乐。  
而这种正向的反馈反过来又促进了互联网软硬件的发展，这使得 Web 开发领域每天都充斥着变化和挑战。  
但总的来说，Web 的发展是循序渐进，有理有据的。个人浅显的认识是 Web 领域可按年代划分为如下大致的几个阶段：

- 1980 年代 太古代
  > Web 的概念还没有出现，但无论是 Linux 还是开源的思想都将会在未来影响着 Web 的发展
- 1990 年代 元古代
  > HTML、URL 和 HTTP Web 的三大基石出现了  
  > 各种浏览器层出不穷  
  > W3C 组织诞生  
  > HTML 的多样渲染强化：Javascript1.0/CSS1.0  
  > 动态网页技术的出现：CGI/PHP/ASP/JSP
- 2000 年代 古生代
  > 三大件(Html，Css，Js)逐渐成熟  
  > 遍地动态页面  
  > 分布式框架出现：J2EE/.Net  
  > 框架横飞的年代：MVC，ORM  
  > REST 模式与 JSON 的诞生(前后端分离的基础)  
  > AJAX/JQuery 诞生（性能/易用性）
- 2010 年代 中生代
  > 前后端分离的 Web 应用遍地开花
  > 前端组件化的技术出现（React/Vue/Angula）  
  > JavaScript 不断升级换代  
  > 自动化测试/运维技术出现 （DevOps 等）  
  > 分布式框架逐步成熟 （SpringBoot/Cloud 和其生态圈）
  > 云的概念诞生（降低了 Web 行业的门槛）  
  > 各种跨平台技术出现  
  > 开源的 Chrome 内核逐步占领市场
- 2020 年代 新生代
  > React/Vue/Angula 如火如荼  
  > 云存储和云服务体系越来越完备  
  > 5G 诞生
- 2030 年代 ???

如此看来 Web 邻域从诞生到现在其实发展了 30 年，Web 技术的迭代速度可以说是前所未有的，这 30 年间所谓沧海变了桑田。  
对此，Web 的世界里，我自己总结了一句话：**五年出新，十年换代。**
